# Часть 6: Контрольные вопросы

## 1) В каком порядке выполняются конструкторы при создании объекта подкласса?
_Ваш ответ:_ Сначала вызывается конструктор родительского класса (явно через super() или неявно), затем конструктор подкласса.

Пример: class B extends A { B() { super(); ... } } - сначала A(), потом B().

## 2) Можно ли переопределить `static` метод? Что такое method hiding?
_Ваш ответ:_ static методы нельзя переопределить, их можно только скрыть - это когда подкласс объявляет static метод с той же сигнатурой, но вызов зависит от типа ссылки, а не объекта.

Пример: A.staticMethod() vs B.staticMethod() — вызывается метод типа ссылки.

## 3) Что произойдет, если в `switch` нет `default` и ни один `case` не совпал?
_Ваш ответ:_Ничего, выполнение просто продолжается после блока switch. 

## 4) Почему в `for-each` нельзя удалять элементы коллекции? Как правильно?
_Ваш ответ:_ Удаление элементов коллекции внутри for-each вызывает ошибку **ConcurrentModificationException**, так как цикл использует встроенный итератор.
Чтобы правильно удалить элементы, можно использовать явный Iterator, метод removelf, итерацию в обратном порядке.

## 5) Чем отличается `break` с меткой от обычного `break`?
_Ваш ответ:_ Обычный break выходит из текущего цикла, а break с пометкой - из внешнего помеченного цикла.

## 6) Что означает `@Override`?
_Ваш ответ:_ Аннотация, которая заставляет компилятор проверить, что метод действительно переопределяет метод родителя. Если нет, то ошибка компиляции.

## 7) Какой модификатор доступа выбрать для видимости в подклассах?
_Ваш ответ:_ Для обеспечения видимости членов класса только в подклассах следует 
использовать модификатор **protected**. Он открывает доступ наследникам, даже если они находятся в других 
пакетах, а также классам внутри того же пакета. Если подклассы всегда находятся в том же пакете, 
подойдет модификатор по умолчанию (package-private).

## 8) Разница между `final` классом и `sealed` классом?
_Ваш ответ:_ final - запрещает наследование полностью. sealed - разрешает наследование только указанным классам.

## 9) Можно ли иметь несколько конструкторов?
_Ваш ответ:_ Да, это перегрузка конструкторов. Они различаются типами, количеством или порядком параметров.
Пример: Car() и Car(String color) - два разных конструктора.

## 10) Что такое ковариантный тип возвращаемого значения?
_Ваш ответ:_  При переопределении метода можно вернуть более конкретный тип, чем в родительском методе.
Пример: Animal reproduce() в родителе, Cat reproduce() в подклассе — возвращает Cat, а не Animal.
